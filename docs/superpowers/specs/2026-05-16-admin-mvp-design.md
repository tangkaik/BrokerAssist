# 管理后台 MVP 设计

## 目标

管理后台是系统管理员工具，不属于普通用户 Web 工作台。第一版只解决三件事：

1. 查看数据库概况：有多少用户、每个用户有多少客户、用户选择了什么行业。
2. 做最小用户维护：查看用户客户列表、重置用户密码。
3. 配置 App 支持的行业：新增、编辑、复制、启用/停用行业，并编辑行业提示词字段。

不做普通用户工作台导航入口，不做复杂图表，不做系统配置中心，不做用户禁用/删除，不做多管理员权限体系。

## 管理员身份

第一版不新增 `is_admin` 字段。

管理员固定为账号名 `administrator`：

- 当前登录用户的 `account` 精确匹配 `administrator` 时，允许访问管理 API。
- 普通注册禁止注册 `administrator`，大小写不敏感拦截，例如 `Administrator`、`ADMINISTRATOR` 都不允许。
- `administrator` 账号由我们手动创建或在数据库里维护。
- 未来如果需要多个管理员，再增加 `is_admin` 字段或管理员表。

## 页面

新增独立页面：

- `web/admin.html`
- 管理后台不挂在普通用户 Web 工作台导航里。
- 页面复用现有登录态；未登录时提示先登录普通 Web，再访问后台。
- 已登录但不是 `administrator` 时显示无权限。

页面包含三个 Tab。

### 1. 用户概览

展示：

- 总用户数
- 总客户数
- 用户列表

用户列表字段：

- 账号
- 昵称
- 当前行业
- 客户数
- 注册时间

点击用户可查看该用户客户列表，只读展示客户姓名、电话、标签、最近更新时间。

### 2. 用户维护

提供：

- 按账号/昵称搜索用户
- 重置密码

重置密码第一版用简单弹窗输入新密码，提交后直接更新。

不做：

- 禁用用户
- 删除用户
- 修改用户行业
- 修改用户资料
- 管理员角色管理

### 3. 行业配置

行业列表展示：

- 行业标识，例如 `generic`、`insurance`、`real_estate`
- 行业名称
- AI 角色名
- 是否启用
- 最近更新时间

支持操作：

- 新增行业
- 编辑行业
- 复制为新行业
- 启用/停用行业

`generic` 通用行业不允许删除，不允许停用。第一版可以不提供删除功能，避免误删老数据。

## 行业字段

界面上按固定字段编辑：

- 行业标识：英文 key，新增后不可改。
- 行业名称：展示给用户看的名称。
- AI 角色名：例如“保险经纪人助手”。
- 客户画像关注点：多行文本，每行一个关注点。
- 缺失信息清单：多行文本，每行一个信息项。
- 拜访建议关注点：一段文本。
- 禁止/边界提示：多行文本，每行一个约束。
- 示例关键词：多行文本，每行一个关键词，用于 AI 助手查询理解和示例。
- 是否启用。

新增行业时可以选择：

- 从空白创建。
- 从已有行业复制配置后修改。

推荐默认流程是“复制已有行业”，降低填写成本。

## 数据设计

第一版新增一张核心表：`industries`。

建议字段：

- `key`
- `label`
- `role_name`
- `enabled`
- `prompt_config`
- `created_at`
- `updated_at`

`prompt_config` 使用 JSON 保存提示词字段：

```json
{
  "summary_focus": [],
  "missing_info": [],
  "advice_focus": "",
  "forbidden_guidance": [],
  "query_examples": []
}
```

界面固定字段编辑，数据库用 JSON 保持灵活。

现有硬编码行业配置继续作为兜底默认值。数据库中没有对应行业时，回退到 `generic`。

## API

新增独立 API 前缀：`/api/v1/admin/*`。

所有接口都需要管理员校验：当前用户账号必须是 `administrator`。

接口：

- `GET /admin/stats`
- `GET /admin/users`
- `GET /admin/users/{id}/customers`
- `PUT /admin/users/{id}/password`
- `GET /admin/industries`
- `POST /admin/industries`
- `PUT /admin/industries/{key}`
- `POST /admin/industries/{key}/clone`
- `PUT /admin/industries/{key}/enabled`

## App 和 Web 的使用方式

App 和普通 Web 工作台只读取启用的行业配置。

用户选择行业时：

- 只展示 `enabled = true` 的行业。
- 已有用户如果绑定了被停用行业，仍保留原行业，不强制迁移。

AI 生成客户画像、拜访建议、客户问答时：

- 根据用户 `industry_key` 读取行业配置。
- 将行业配置填入现有通用 prompt 模板。
- 不为每个行业维护一整套大 prompt。

## 测试重点

- 普通用户不能访问管理 API。
- `administrator` 可以访问管理 API。
- 注册时禁止 `administrator` 及其大小写变体。
- 行业新增、编辑、复制、启用/停用生效。
- App/Web 读取行业时只展示启用行业。
- 保存拜访记录后生成画像/建议时能使用数据库行业配置。

## 暂不做

- 多管理员。
- 后台独立登录系统。
- 系统配置中心。
- 用户禁用/删除。
- 行业删除。
- 图表化仪表盘。
- 完整审计日志。
