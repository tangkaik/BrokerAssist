## 三、问题清单（优先级排序）

> ✅ = 已完成修复 / 🚧 = 进行中 / ⏳ = 待处理

---

### P0 - 立即修复

#### 1. 后端：`ai_routes.py` 路由层业务泄露

**状态**: ✅ 已完成

**改进内容**:
- AI 问答逻辑从 `ai_routes.py` (~800行) 抽取到 `ai_service.py`
- 路由层从 ~92 行缩减到 ~92 行，仅保留依赖注入和调用
- 业务规则（两个月未联系、优先级排序、地域问题、图片问答）集中在 `AIService`

**新分层链**:
```
ai_routes.py  ← 路由（薄）
       ↓
AIService     ← 主业务编排
       ↓
KimiClient   ← AI 客户端
```

---

#### 2. 后端：JSON 文件存储并发不安全

**状态**: ⏳ 待处理

**位置**:
- `/app/services/customer_advice_store.py`
- `/app/services/record_image_store.py`
- `/app/services/record_image_analysis_store.py`

**问题**: 使用 `threading.Lock`，但 FastAPI 使用异步。高并发下可能出现竞态条件。

```python
_STORE_LOCK = Lock()  # threading.Lock 在 async 中不安全
```

**建议**:
- 方案 A: 改用 `asyncio.Lock`
- 方案 B: 迁移到数据库表

---

#### 3. 后端：图片分析逻辑独立

**状态**: ✅ 已完成

**改进内容**:
- OCR/图片识别逻辑从 `record_service.py` 抽取到 `record_image_analysis_service.py`
- `RecordImageAnalysisService` 职责清晰：
  - `_build_analysis_prompt` - prompt 模板管理
  - `analyze_record_image` - 调用视觉模型
  - `get_record_analysis_map` / `delete_record` - 结果读取

---

#### 4. 后端：记录媒体服务分离

**状态**: ✅ 已完成

**改进内容**:
- 记录图片管理从 `record_service.py` 抽取到 `record_media_service.py`
- `RecordMediaService` 接管：
  - `save_images` / `replace_images` / `delete_record_assets` - 图片文件管理
  - `build_record_item` - 组装 `RecordItem.images`

**新分层链**:
```
record_routes.py   ← 路由（薄）
        ↓
RecordService      ← 主业务编排
        ↓
RecordMediaService        ← 媒体编排
        ↓
RecordImageAnalysisService ← OCR/识别
        ↓
RecordImageStore / RecordImageAnalysisStore ← 文件+JSON存储
```

### P1 - 短期优化

#### 7. 后端：JWT 实现非标准

**状态**: ⏳ 待处理

**位置**: `/app/core/security.py`

**问题**: 使用自定义 base64 + HMAC 实现 JWT，未使用 `python-jose` 或 `PyJWT` 库。

**风险**:
- 非标准实现可能存在安全漏洞
- 难以与其他系统集成

**建议**: 使用 `python-jose`

---

#### 8. 后端：密码哈希非标准

**状态**: ⏳ 待处理

**位置**: `/app/core/security.py`

**问题**: 使用自定义 PBKDF2 实现，未使用 `bcrypt` 或 `argon2`。

**建议**: 使用 `passlib[bcrypt]`

---

#### 9. 后端：无数据库迁移工具

**状态**: ⏳ 待处理

**位置**: `/app/db/init_db.py`

**问题**: 使用 `Base.metadata.create_all()` 意味着无法 track 迁移历史、无法回滚。

**建议**: 引入 Alembic（项目已有 `alembic.ini`，未启用）

---

#### 10. 后端：CORS 开放

**状态**: ⏳ 待处理

**位置**: `/app/main.py`

```python
CORSMiddleware,
allow_origins=["*"],  # 生产环境风险
```

**建议**: 环境变量控制允许的域名列表

#### 11. 移动端：配置分散

**位置**: `config.dart` + `api_config.dart`

## 七、本轮重构总结（2026-04-19）

### 已完成

| 文件 | 改进 |
|------|------|
| `ai_routes.py` | ~800行 → ~92行，业务逻辑迁至 `AIService` |
| `ai_service.py` | 新增，集中所有 AI 问答规则 |
| `record_service.py` | 瘦身，仅保留主业务编排 |
| `record_image_analysis_service.py` | 新增，OCR/图片识别独立 |
| `record_media_service.py` | 新增，媒体文件管理独立 |

### 工程价值

1. **换模型/prompt 不再碰路由** - 改 `AIService` 即可
2. **换 OCR/视觉模型 不再碰记录主流程** - 改 `RecordImageAnalysisService` 即可
3. **图片管理逻辑与业务逻辑解耦** - `RecordMediaService` 可被复用
4. **每层可独立测试** - service 层与路由层分离
