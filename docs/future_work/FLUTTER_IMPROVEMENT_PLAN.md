# BrokerAssist Flutter 移动端改进计划

## 一、核心功能补全（影响日常使用）

### 1.1 客户编辑功能
**问题**：现在只能看客户，不能编辑地址、手机、标签

**涉及页面**：
- `customer_detail_page.dart` — 增加"编辑"按钮
- `edit_customer_page.dart` — 新建编辑页面（复用 create 表单结构）
- `api.dart` / `customer_api.dart` — PUT 接口已存在，前端未调用

**改动**：
1. 新建 `edit_customer_page.dart`（参考 create 页面）
2. `CustomerDetailPage` 加编辑按钮，导航到编辑页
3. 编辑页表单填充现有数据，保存调用 PUT `/customers/{id}`
4. 保存后返回详情页并刷新

### 1.2 记录编辑/删除
**问题**：只能添加记录，错误记录无法修改

**涉及页面**：`customer_detail_page.dart`

**改动**：
1. 每条记录卡片加编辑/删除按钮（icon button）
2. 长按弹出菜单（编辑/删除）
3. 编辑弹窗复用创建记录逻辑，删除调用 DELETE

### 1.3 客户列表分页
**问题**：超过 100 个客户无法加载更多

**涉及页面**：`customer_list_page.dart`

**改动**：
1. 滚动到底部自动加载下一页（`ScrollController` 监听）
2. 或加"加载更多"按钮
3. API 层传 `page` + `page_size` 参数

---

## 二、AI 聊天体验提升

### 2.1 聊天记录本地持久化
**问题**：App 重启后 AI 聊天记录全丢

**改动**：
1. 用 `shared_preferences` 或 SQLite 保存聊天历史（JSON 格式）
2. `AIChatPage` 启动时加载历史记录
3. 按用户隔离（每个用户自己的历史）

### 2.2 搜索历史功能
**问题**：只有 5 个硬编码的推荐问题

**改动**：
1. 保存用户历史提问到本地
2. 显示最近 10 条历史提问，点击快速重发
3. 清空历史按钮

---

## 三、UI/UX 改进

### 3.1 客户头像
**问题**：列表所有客户都是默认头像

**改动**：
1. 客户 initials 头像（姓名首字，颜色根据姓名 hash 生成）
2. 如有头像 URL 则显示真实头像

### 3.2 骨架屏（Skeleton）
**问题**：加载只有 spinner，体验单调

**改动**：
1. 客户详情页加载时显示骨架占位
2. AI 回答加载时显示"正在思考..."动画

### 3.3 错误状态优化
**问题**：错误只显示"加载失败"

**改动**：
1. 显示具体错误信息（网络断开/服务器错误/超时）
2. 配重试按钮
3. 空列表状态加插画或引导文案

---

## 四、代码质量

### 4.1 `home_page.dart` 拆分（约 870 行）
拆分为独立组件：
- `RecordingSection` — 录音区域
- `TranscriptionSection` — 转写结果区域
- `PlaybackSection` — 播放控制区域
- `DraftActionsBar` — 底部操作按钮区

### 4.2 API 层统一错误处理
**问题**：各页面独立处理错误，代码重复

**改动**：
1. `ApiClient` 统一拦截错误，做 toast 提示
2. 401 自动跳转登录页
3. 减少各页面 try-catch 重复代码
