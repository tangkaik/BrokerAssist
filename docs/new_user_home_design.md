# 新人用户首页设计方案

2026-05-05

## 设计原则

- 填空但不堆垃圾信息 —— 每一块都有信息价值
- 不增加虚假交互 —— 不搞"点击完成"的伪进度条，真实流程会自然推进
- 预告而不说教 —— 让用户知道接下来会发生什么，而不是教学
- 一次性 —— 老用户看不到这些，只在新用户首页出现

## 布局结构（从上到下）

```
┌─────────────────────────────────────────┐
│  1. 行业引导卡片                         │
│     根据所选行业展示不同的引导文案        │
├─────────────────────────────────────────┤
│  2. 3 步流程示意                        │
│     横向三格：记录 → 归档 → AI 整理       │
├─────────────────────────────────────────┤
│  3. 开始记录卡片                         │
│     主操作入口，含录音/文字/图片入口       │
├─────────────────────────────────────────┤
│  4. 一句话实操提示                       │
│     基于行业的一句简短指引                │
├─────────────────────────────────────────┤
│  5. AI 入口卡片                         │
│     引导用户尝试 AI 助手                  │
└─────────────────────────────────────────┘
```

## 各卡片详细设计

### 1. 行业引导卡片（已有）

继承当前 `IndustryGuideBanner`，文案按行业区分：

- 通用："开始建立你的客户工作台" / "先记录一次沟通，系统会自动整理客户画像、待补信息和下一步动作。可以在"我的"页面随时调整行业方向。"
- 特定行业："已为你启用{行业名}行业视角。首页和建议会按这个方向组织，后续可在"我的"页面调整。"

状态：**已实现。**

### 2. 三步流程示意（新增）

无交互、纯展示。让新用户理解系统如何工作。

视觉形式：三个小卡片横向排列，中间用箭头连接。

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   🎙️               👤               🤖                   │
│   记录              归档              AI 整理              │
│                                                          │
│   语音/文字/图片    创建客户          自动生成画像          │
│   记一次沟通        挂到客户名下      随时查看建议          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

每个格子结构：

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  图标     │ →  │  图标    │ →  │  图标    │
│  标题     │    │  标题    │    │  标题    │
│  描述     │    │  描述    │    │  描述    │
│  描述     │    │  描述    │    │  描述    │
└──────────┘     └──────────┘     └──────────┘
```

具体内容：

| 步骤 | 图标 | 标题 | 描述 1 | 描述 2 |
|------|------|------|--------|--------|
| 1 | mic_none | 记录 | 语音/文字/图片 | 记一次沟通 |
| 2 | person_add | 归档 | 创建客户 | 挂到客户名下 |
| 3 | auto_awesome | AI 整理 | 自动生成画像 | 随时查看建议 |

视觉规格：
- 整体容器白色背景，轻微边框，圆角 8px
- 三格等宽，使用 `Row` + 3 个 `Expanded`
- 箭头用 `Icons.chevron_right` 或 `Icons.arrow_forward`，放在格与格之间
- 每格内部垂直居中排列：图标 → 标题 → 两行描述
- 图标 28px，在圆形浅色底上
- 标题 14px 加粗，描述 11px 灰色
- 若屏幕过窄（如 320px），可改为竖排三行

Widget 名：`OnboardingFlowSteps`

### 3. 开始记录卡片（已有）

继承当前 `StartRecordCard`，新人版文案：
- 标题："记录第一次沟通"
- 副标题："录音、文字或图片都可以，先记下来再归档到客户。"
- 主按钮："开始记录"（调用 `onStartRecord`）
- 次按钮："直接新建客户"（调用 `onCreateCustomer`）

状态：**已实现。**

### 4. 一句话实操提示（新增）

轻量到不能再轻量。一行底色 + 一行文字。

```
┌─────────────────────────────────────────┐
│  💡 试试看：每次见完客户，打开录音说    │
│     60 秒，AI 自动帮你整理要点和下一步   │
└─────────────────────────────────────────┘
```

视觉：浅蓝/浅灰底色，左侧灯泡图标，右侧 1-2 行文字。无按钮、无交互。

基于行业变化：

| 行业 | 提示文案 |
|------|---------|
| 保险 | 每次见完客户，打开录音说 60 秒，AI 自动帮你整理要点和下一步建议 |
| 教育 | 记录每个家长关心的科目和薄弱点，AI 自动归类并生成跟进建议 |
| 房产 | 记录客户偏好和预算范围，AI 自动整理并提醒匹配的房源方向 |
| 法律 | 记录咨询要点和案件类型，AI 自动整理案情摘要和待办事项 |
| 其他 | 把每次客户沟通记下来，AI 自动回顾并帮你判断下次跟进时机 |
| 通用 | 把每次客户沟通记下来，AI 自动回顾并帮你判断下次跟进时机 |

Widget 名：`IndustryQuickTip`

## 不放入新人首页的内容

- 客户列表、跟进建议、统计数字 —— 用户还没有客户
- 图片轮播、功能介绍长文 —— 用户不想看的垃圾信息
- 勾选清单、"已完成"进度条 —— 与实际流程有冲突
- 客户画像预览 —— 还没有内容可展示
